# AlertsDLX — Architecture Overview

High-level reference for the AlertsDLX WordPress plugin: four themed alert blocks, shared editor/frontend layers, and how PHP, JavaScript, and CSS are loaded.

---

## What the plugin does

AlertsDLX provides **four Gutenberg blocks**, each modeling a different design system:

| Theme folder   | Block name                      | `alertGroup` |
|----------------|---------------------------------|--------------|
| `bootstrap/`   | `mediaron/alerts-dlx-bootstrap` | `bootstrap`  |
| `chakraui/`    | `mediaron/alerts-dlx-chakra`    | `chakra`     |
| `material/`    | `mediaron/alerts-dlx-material`  | `material`   |
| `shoelace/`    | `mediaron/alerts-dlx-shoelace`  | `shoelace`   |

All four blocks share the same **attribute shape** (colors, layout, icon/image, button, dismiss, editorial flags) and the same **frontend render path** in PHP. They differ mainly in **theme-specific variants**, **block styles** (WP core style presets), **icon sets**, and **SCSS**.

Blocks use **InnerBlocks** for alert body content (description). Title, button, and icon are block attributes.

## PHP loading and structure

### Bootstrap

`alerts-dlx.php` defines constants, loads `lib/autoload.php` (Composer, namespace `DLXPlugins\AlertsDLX\` → `php/`), and on `plugins_loaded` runs:

1. **`Options::run()`** — Plugin options (headline tag, enabled block themes, debug flag) via WordPress Options API (`alerts-dlx` option key).
2. **`Admin::run()`** — Settings submenu under **Settings → AlertsDLX**, React admin UI, AJAX save/retrieve/reset.
3. **`Blocks::run()`** — Block registration, render callbacks, editor script localization, shortcode `[alertsdlx]`.

### Main PHP classes

| Class        | Role |
|--------------|------|
| `Functions`  | Helpers: paths/URLs, attribute sanitization, KSES allowlists, theme color palette |
| `Options`    | Defaults and getters for site-wide plugin settings |
| `Admin`      | Settings page shell + AJAX for the React admin app |
| `Blocks`     | Register blocks from `build/`, `frontend()` render for all four blocks, enqueue/register assets |

### Block registration (server)

On `init`, `Blocks` registers types from **`build/js/blocks/`** using `build/blocks-manifest.php` (generated by `@wordpress/scripts`).

Render callbacks are attached via `block_type_metadata_settings` so all four blocks call the same **`Blocks::frontend()`** method. The `alertGroup` attribute selects which theme CSS/classes apply.

Disabled themes (from admin settings) are hidden from the inserter via `register_block_type_args` → `supports.inserter = false`.

### Frontend rendering

`Blocks::frontend()`:

- Sanitizes attributes
- Outputs semantic HTML (`figure` + `section`, shared BEM-style classes like `alerts-dlx-alert`, `alerts-dlx-{group}`)
- Uses **`alerts_dlx_the_content`** filter chain (not `the_content`) for InnerBlocks HTML
- Prints the matching **theme stylesheet** in `wp_footer` (`alerts-dlx-{theme}-styles` + Lato font)
- Respects editorial-only and dismiss-cookie behavior

Headline tag for the title comes from **`Options::get_headline_style()`** (site setting: `h1`–`h6` or `div`).

## JavaScript loading and structure

### Build pipeline

Two webpack configs (see `package.json`):

1. **`wp-scripts build`** — Compiles `src/index.js` → `build/index.js` + `build/index.css`, copies `block.json` files to `build/js/blocks/`, generates `blocks-manifest.php`.
2. **`webpack` (`build:dist`)** — Compiles theme SCSS, admin React app, dismiss script, and web fonts into **`dist/`**.

### Editor entry: `src/index.js`

Imports all four block registrations and cross-block **editor plugins**:

- `advanced-inner-blocks.js` — Slot fill for flexible InnerBlocks toggle
- `editorial-only.js` — Editorial / read-only inspector (HOC on `editor.BlockEdit`)
- `custom-colors.js` — Custom color panel when `alertType === 'custom'` (HOC, Styles tab)
- `icon-image-selector.js` — Icon vs image source (Slot fill)

Also sets the **AlertsDLX** block category icon.

### Per-block structure

Each theme folder contains:

- **`block.json`** — Attributes, block styles, `editorScript` / `editorStyle` handles
- **`index.js`** — `registerBlockType`, transforms between themes, `save` returns `<InnerBlocks.Content />`
- **`edit.js`** — Inspector panels, slots, delegates canvas UI to shared **`BlockMain`**

### Shared editor components (`src/js/blocks/components/`)

| Component        | Purpose |
|------------------|---------|
| `BlockMain`      | Canvas: title RichText, icon/image, InnerBlocks, button, close button |
| `IconPicker`     | SVG icon selection / custom SVG input |
| `AlertButton`    | Button text/URL controls in the canvas |
| `CloseButtonIcons` | Theme-specific close icons |
| `unit-picker`    | Max-width unit control |
| `icons/*`        | Per-theme logos and icon libraries (BootstrapIcons, ChakraIcons, etc.) |

### Editor plugins pattern

Two extension patterns:

1. **`createHigherOrderComponent` + `addFilter('editor.BlockEdit', …)`** — Wraps all AlertsDLX blocks (e.g. custom colors, editorial-only).
2. **`registerPlugin` + `<Fill name="…">`** — Injects controls into **Slots** declared in each block’s `edit.js` (e.g. `alertsDLXSettingsPanelEnd`, `alertsDLXAppearancePanelEnd`).

### Localized editor data

`Blocks::register_block_editor_scripts()` registers `alerts-dlx-block` and exposes **`alertsDlxBlock`**:

- `headlineStyle`, `enabledBlockStyles`, `colorPalette`, `defaultImage`, capability flags, font stylesheet URL

Referenced in editor JS as a global (`alertsDlxBlock`).

## CSS loading and structure

### Source (`src/scss/`)

| File / folder              | Built to (dist)                    | Used for |
|----------------------------|------------------------------------|----------|
| `common.scss`              | `alerts-dlx-common.css`            | Shared base |
| `block-editor.scss`        | `alerts-dlx-block-editor.css`      | Editor chrome |
| `bootstrap/styles.scss`    | `alerts-dlx-bootstrap-styles.css`  | Bootstrap block |
| `chakra/styles.scss`       | `alerts-dlx-chakra-styles.css`     | Chakra block |
| `material/styles.scss`     | `alerts-dlx-material-styles.css`   | Material block |
| `shoelace/styles.scss`     | `alerts-dlx-shoelace-styles.css`   | Shoelace block |
| `fonts/lato.scss`, `roboto.scss` | Font CSS bundles             | Typography |

`src/index.js` (wp-scripts) also emits **`build/index.css`** for component-level editor SCSS (IconPicker, AlertButton, unit-picker).

### Editor

- `block.json` references `alerts-dlx-block` script and combined editor style handles.
- PHP registers and chains: `alerts-dlx-block-editor` (dist) → `alerts-dlx-block-editor-styles` (build) + per-theme dist CSS.

### Frontend

Theme CSS is **registered** in PHP and **printed in `wp_footer`** when a block renders (not globally enqueued on every page). Class names and CSS variables follow the pattern `--alerts-dlx-{alertGroup}-color-*`.

## Admin settings (separate from blocks)

- **Page:** Settings → AlertsDLX (`php/Admin.php`)
- **UI:** React app in `src/react/Settings/` → `dist/alerts-dlx-admin-settings.js` + `dist/alerts-dlx-admin-style.css`
- **Persistence:** WordPress Options API via AJAX (`alerts_dlx_save_settings`, etc.)
- **Settings affect blocks:** headline HTML tag, which of the four themes appear in the block inserter

---

## Shared concepts across all four blocks

- **Attributes:** `alertType`, `variant`, `mode`, colors, layout (`maximumWidth`, `baseFontSize`), structure toggles (`iconEnabled`, `titleEnabled`, …), button fields, `uniqueId`, `alertGroup`, icon/image (`icon`, `iconSource`, `imageId`), dismiss (`closeButtonEnabled`, `closeButtonExpiration`), editorial flags
- **Content model:** `alertTitle` + InnerBlocks body; legacy `alertDescription` migrated to InnerBlocks in the editor
- **Dynamic blocks:** Saved markup is InnerBlocks only; PHP `render_callback` builds full alert HTML
- **Shortcode:** `[alertsdlx]` reuses the same render pipeline with attribute mapping in `Blocks::shortcode()`
- **Dismiss:** Frontend cookie `alerts-dlx-{uniqueId}`; `src/js/dismiss/` → `dist/alerts-dlx-dismiss.js`

## Key files to start from

| Task | Start here |
|------|------------|
| Change frontend HTML | `php/Blocks.php` → `frontend()` |
| Add block attribute | All four `src/js/blocks/*/block.json` + `frontend()` sanitization |
| Editor inspector UI | Theme `edit.js` and/or `src/js/blocks/plugins/` |
| Theme appearance | `src/scss/{theme}/styles.scss` |
| Site-wide settings | `php/Options.php`, `src/react/Settings/` |
| Register/build blocks | `src/index.js`, `npm run build` |

## Build commands

```bash
npm run build    # wp-scripts (build/) + webpack (dist/)
npm run start    # wp-scripts watch for block editor development
```

After changing `block.json` or `src/index.js`, run **`npm run build`** before testing in WordPress.
